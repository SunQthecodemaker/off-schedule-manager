# 연차 시스템 개편 작업 상세 내역 (UI 및 데이터 연산 구조 업데이트)

## 반영 일자: 2026-03-04
이 문서는 메인 PC 외 다른 PC에서 작업 내역을 파악하거나 배포를 확인할 수 있도록 남기는 마일스톤 로그입니다.

### 1. 연차 모달(팝업창) 폐기 및 인라인 화살표 네비게이션 도입
* **기존 문제:** `연차 현황` 테이블에서 특정 직원의 이름을 클릭하면 화면에 큰 '최근 3년 연차 내역' 팝업창이 떠서, 다시 닫은 후 다른 직원을 봐야 하는 등 조작이 번거로웠습니다.
* **해결 방안:** 메인 테이블 직원명 우측 연차 박스 컨테이너 바로 옆에 `◀` `▶` 화살표 버튼을 추가해, **표 안에서 직접 이전 연차 주기 / 다음 연차 주기**를 넘겨볼 수 있게 인라인 네비게이션으로 전면 교체했습니다.
* 관리자 뷰(`management.js`) 및 직원 뷰(`employee-portal-final.js`) 양쪽에 동일한 규칙으로 적용해 통일감을 주었습니다.
* `index.html`에 존재하던 기존의 무거운 `leave-history-modal`, `employee-leave-history-modal` HTML 블록은 깔끔하게 삭제되었습니다.

### 2. 가로 스크롤 제거 및 한 줄 날짜 라벨을 통한 디자인 최적화
* **기존 문제:** 1년에 사용하는 연차가 15개가 넘어갈 때 박스가 옆으로 너무 길어지면 일부 스크롤바가 생기고 불편했습니다. 또한 표기된 기준일 라벨이 3줄로 되어 있어 세로 폭도 너무 컸습니다.
* **디자인 최적화:** 
  1. `flex-wrap: wrap` CSS 속성 추가. 박스들이 컨테이너를 넘어가면 아래로 자연스럽게 단락을 나누며 줄바꿈하여 한눈에 보이도록 수정했습니다.
  2. 주기 기간 라벨을 "24.03.01 ~"처럼 컴팩트하게 **한 줄**로 통일했습니다. 이렇게 하여 불필요한 세로/가로 공간 차지를 줄이고 리스트의 높이를 일정하게 유지했습니다.

### 3. '당겨쓰기' 로직의 톱니바퀴 이관(Seamless Transition) 렌더링
* 가장 까다로운 데이터 표출 케이스를 완벽하게 제어했습니다.
* **올해 당겨쓰기 표출 패턴:** 직원이 당해년도(N년도)에 다음 년도 연차 1개를 미리 당겨쓰면 렌더링 방식이 아래의 두 가지 모습에 따라 똑똑하게 모습을 달리합니다.
  - **N년도 화면 (예: 24년 주기 뷰):** 연차 박스 가장 맨 끝칸 이후에 빨간색 '초과 박스'로 붙어서 "올해 1개 오버해서 썼다"고 표시.
  - ▶ 버튼 클릭 시 -> **N+1년도 화면 (예: 25년 주기 뷰):** 방금 전 초과 사용했던 박스는 이 주기에서는 사실상 "미리 선차감된 1번 박스"가 되므로, 이번에는 박스 배열의 **가장 첫 칸**에 보라색(수동차감/당겨쓰기)으로 자리잡도록 데이터 재조립 로직을 마련했습니다.

### 4. 과거 엑셀 데이터 임포트 도구 제작
* 시스템 도입(25년도) 이전의 엑셀 수기 관리 연차 사용 내역을 Supabase DB의 `leave_requests`에 일괄 승인건으로 업로드할 수 있는 임시 관리자용 툴을 마련했습니다.
* 위치: `import-leaves.html` (별도 URL에 접속해 기존 엑셀 데이터를 복사&붙여넣기하면 부서/직원 이름별로 식별 후 인서트)

### 주요 수정 파일 요약
1. `index.html`: 구형 모달 마크업 청소.
2. `management.js`:
   - `window.periodOffsets` 상태 변수 딕셔너리로 전직원의 현재 열람 주기를 메모리 수준에서 관리.
   - `changeLeavePeriod()` 트리거 함수 신규 생성.
   - 당겨쓰기된 배열을 현재-과거 간 적절하게 복제/병합하여 `usedDates`를 산출.
3. `employee-portal-final.js`: 
   - 관리자와 마찬가지로 단일 사용자에 대한 `periodOffset` 상태 관리 도입.
   - `renderEmployeeLeaveGrid()` 함수로 인라인 뷰 마크업 및 박스 CSS, wrap 등 관리자단과 동일하게 렌더링 최적화.
4. `import-leaves.html`: 엑셀 복붙용 임포터 스크립트 작성.
